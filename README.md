# Packet-Tracer-Using-Telnet-and-SSH

## Project Overview
This project demonstrates how to remotely access a router using Telnet and SSH in Cisco Packet Tracer. The activity includes verifying connectivity, configuring PCs for remote access, and testing secure SSH access.

## Objectives
- Verify PC IP addressing
- Test connectivity to the HQ router
- Access a router remotely using Telnet and SSH

## File Structure
- `PacketTracerFiles/`: Contains the Packet Tracer `.pkt` file
- `Instructions/`: Step-by-step instructions for each part
- `Images/`: Screenshots of IP configurations and SSH session
- `Notes/`: Addressing tables and supporting documentation

## Usage
1. Open `TelnetSSH.pkt` in Cisco Packet Tracer.
2. Follow the instructions in the `Instructions/` folder.
3. Verify connectivity and attempt Telnet and SSH connections.
